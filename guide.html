Project Guide

Build, publish, and present the project

This page collects the execution steps for Power BI, GitHub Pages, and the key repo assets so you do not need to jump through raw markdown files while previewing the site.

Power BI

What to build in the report

Page 1: Executive Overview

KPI cards, top repositories, top developers, impact tiers, and language distribution.

Page 2: Repository Deep Dive

Scatter plot, lifecycle distribution, company contribution, and a drillable detail table.

Page 3: Data Quality and Model Design

EER diagram, matching constraints, cleaning logic, and future schema extensions.

Detailed bilingual visual specs are in docs/powerbi-visual-spec.md. Layout storyboard is in docs/powerbi-report-blueprint.md.

Launch

How to preview and publish the website

Local preview

Run bash preview-site.sh in the project folder, then open http://localhost:8000.

Power BI embed

Publish the Power BI report, copy the public embed URL, and paste it into app.js.

GitHub Pages

Push the repo to GitHub and publish from the root branch folder using the steps in the launch guide.
