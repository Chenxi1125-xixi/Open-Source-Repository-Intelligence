Data Model

How the database is structured

Tables

The schema includes developers, repositories, repository metrics, languages, and companies.

Why this matters

The relational design separates profile information, repository metadata, and performance metrics so the analysis is easier to query and explain.

Important source limitation

The repo metadata file has 14,644 repositories, but only about 200 repositories match the separate GitHub users file. Because of that, the live dashboard focuses on the full repository source.

Findings

Main takeaways from the analysis

Visibility is concentrated

A small number of organization-backed repositories drive most of the stars and forks in the sample.

Python dominates the sample

Most repositories in the curated dataset use Python, which also highlights the need for future language expansion.

Most projects are early stage

The impact tier breakdown shows a long tail of repositories with low visibility and only a few standout projects.

README usage is high

README coverage is strong in the repository source, which helps explain why many projects still look documented even when their engagement is low.

Implementation

How the built-in BI works

What it is

This is not actual Power BI embedded in the page. It is a custom web dashboard built with HTML, CSS, JavaScript, and a JSON data file.

Why use it

It makes the website self-contained, so visitors can explore the results without leaving the page or needing a published BI link.

Power BI still fits

You can still build a formal Power BI version separately for class, interviews, or portfolio comparison using the same cleaned dataset.
